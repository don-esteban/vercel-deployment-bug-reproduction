<Intro>

Red Hat Enterprise Linux Server mit Standard oder Premium Support ist für Produktionsumgebungen geeignet. Die beiden Editionen können auf physischen und virtuellen Systemen oder auch in der Cloud genutzt werden. Red Hat Enterprise Linux Server Entry Level (Self-Support) ist nur bedingt für physische Testumgebungen geeignet und enthält keinen Support.

</Intro>

## Server Übersicht

Aktuell sind die Versionen [RHEL 9](https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/9) und [RHEL 8](https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/8) unter Full Support, siehe auch [Red Hat Enterprise Linux Life Cycle (englisch)](https://access.redhat.com/support/policy/updates/errata/).

<ArrowList>

- [Red Hat Enterprise Linux Server Entry Level (Self-Support)](#entry-level) <br />
Enthält RHEL Server nur für **physische Testsysteme** (maximal 2 Sockets) und man hat keinen Supportanspruch. Es ist nur das Add-on Satellite verfügbar.
- [Red Hat Enterprise Linux Server Standard](#standard) <br />
Enthält RHEL Server (Physical/Virtual) mit Standard Support (Reaktionszeiten zu üblichen Geschäftszeiten) und Red Hat Enterprise Linux Atomic Host. Angeboten werden auch Bundles und dazu buchbare Add-ons für Storage (Resilient Storage), High Availability (HA) und Management (Satellite).
- [Red Hat Enterprise Linux Server Premium](#premium) <br />
Enthält RHEL Server (Physical/Virtual) mit Premium Support (bessere Reaktionszeiten), Red Hat Enterprise Linux Atomic Host und Extended Update Support. Angeboten werden auch Bundles und dazu buchbare Add-ons für Storage (Resilient Storage), High Availability (HA) und Management (Satellite).

</ArrowList>

## Preise

Red Hat Enterprise Linux Server wird pro 2er Sockets oder 2er virtuellen Instanzen lizenziert. Erhältlich mit Self-Support oder [Support (SLA) Subscription (Standard oder Premium)](/hersteller/red-hat/lizenzierung/#vergleich-von-standard-und-premium-support). Self-Support hat **keinen Anspruch** auf Support. Self-Support Subscription kann nicht mit Wartung (Premium oder Standard) kombiniert werden. Dazu gibt es [Add-ons](#add-ons). Mehr in unserem Guide [RHEL Kauf und Lizenzierung](/hersteller/red-hat/rhel-lizenzierung/).

## Add-ons

Add-ons werden nach der Anzahl an vorhandenen RHEL Server Subscription des jeweiligen Servers lizenziert. Entweder werden auf dem Server alle Subscriptions mit Add-on(s) ausgestattet oder keine. Für RHEL Server Entry Level steht nur Satellite zur Verfügung.


### Satellite

Zugriff auf [Red Hat Satellite](/hersteller/red-hat/satellite/) (Managementplattform) und [Red Hat Insights](https://www.redhat.com/de/technologies/management/insights). Installation auf jedem verwalteten Gerät.

<Products bkey="red-hat" tags={["addon-smart"]}/>

*Hinweis: Smart Management wurde in Satellite umbenannt. Der Funktionsumfang ist unverändert.

### Resilient Storage

Für RHEL Standard und Premium. Datenpool, der allen Servern in der Gruppe zur Verfügung, auch bei Serverausfall. Ermöglicht über das Netzwerk Zugriff auf das gleiche Block-Storage-Gerät. Beinhaltet High Availability (HA).

<Products bkey="red-hat" tags={["addon-storage"]}/>

### High Availability

Für RHEL Standard und Premium. Für Failover Szenarien. 16 Knoten (physische Systeme oder virtuelle Instanzen) werden maximal unterstützt.

<Products bkey="red-hat" tags={["addon-ha"]}/>

## Funktionen

Red Hat Enterprise Linux bietet eine stabile Grundlage für Ihre Open-Hybrid-Cloud-Infrastruktur – von Bare Metal über virtuelle Maschinen (VMs), Edge Computing, Private Cloud und Public Clouds. Wir zeigen Ihnen einen kleinen Auszug aus den [RHEL Inhalten und Funktionen](https://www.redhat.com/de/technologies/linux-platforms/enterprise-linux):


- Stabile Plattform für alle Hybrid Cloud-Deployments
- Unterstützt die Entwicklung für containerbasierte (Cloud Computing-)Umgebungen
- Ausführung von Paketen und Applikationen als Container (Red Hat Enterprise Linux Atomic Host)
- Ermöglicht automatisierte Edge-Erweiterungen
- Automatisierung von Aufgaben, Workloads und Administrationen
- Erstellen von edge-optimierte Betriebssystem-Images und minimiert so Workload-Unterbrechungen
- Integrierte Sicherheits-Features wie SELinux (Security-Enhanced Linux) und MAC (Mandatory Access Controls)


Um einen Eindruck über die Möglichkeiten mit Red Hat zu gewinnen, lesen Sie auch die [Unternehmensprozesse und Entwicklung mit Red Hat](https://www.redhat.com/de/success-stories).
