<Intro>

JetBrains TeamCity ist ein Java basiertes Buildmanagement Tool mit einem CI (Continuous Integration) Server. Es enthält die Funktionalität der Produkte IntelliJ IDEA und ReSharper sowie Analyse und Kontrolle für Java- und .NET-Code.

</Intro>

## Preise

<ArrowList>

- Lizenziert nach Server inklusive Build Agents. Ein Upgrade der Anzahl der Build Agents ist jederzeit möglich.
- Bei Neukauf erhalten Sie neben der Lizenz eine jährliche Subscription für Upgrades. Die Lizenz können Sie auch ohne aktive Subscription weiterhin auf dem letzten lizenzierten Versionsstand nutzen.

</ArrowList>

## Funktionen


- Unterstützt Java, .NET and Ruby
- Integrierbar mit Eclipse, IntelliJ IDEA, Visual Studio
- Integrierbar in die Cloud wie Amazon EC2, Microsoft Azure und VMware vSphere
- Unterstützt Key Tools
- Unlimitierte Anzahl an Build Konfigurationen
- Code-Refactoring und -Generierung
- Code Analyse
- Reports und Status und interagiert über REST API
- User Management, z.B. Rollen- und Gruppenzuweisung und Autorisierungskontrolle
- Run & Debug
- Unit Tests
- Versionskontrolle
- Priorisierter E-Mail Support
- Docker-Support
- Erweiterbar mit Plugins


_*Eine vollständige Funktionsübersicht finden Sie auf der [JetBrains TeamCity Funktionsübersichts Seite](https://www.jetbrains.com/de-de/teamcity/features/)._

## Was ist ein Build Agent?

Bei JetBrains ist ein Build Agent eine Software, die auf die Befehle des TeamCity-Servers wartet und die eigentlichen Build-Prozesse startet. Ein Agent kann jeweils nur einen Build ausführen. Falls Sie mehrere Builds gleichzeitig ausführen wollen, benötigen Sie entsprechend die Anzahl an Build Agent Lizenzen. Mehr dazu finden Sie auf der [JetBrains TeamCity Dokumentationsseite (Englisch)](https://www.jetbrains.com/help/teamcity/build-agent.html).
