IBM Spectrum Protect unterstützt physikalische Fileserver, virtuelle Umgebungen und gängige Applikationen wie [Microsoft Exchange](/hersteller/microsoft/exchange/), [Lotus Domino/Notes](/hersteller/hcl/lotus-domino-notes/), [Microsoft SQL Server](/hersteller/microsoft/sql-server/), [Oracle Database](/hersteller/oracle/).


## Suites oder Einzelprodukte?

Sie können Spectrum Protect als Programmpaket mit allen verfügbaren Modulen oder einzeln *à la Carte* lizenzieren.

**Suites** enthalten alle Spectrum Protect [Einzelprodukte](#ibm-spectrum-protect-einzelprodukte). Lizenziert werden Suites nach [zu sichernder Speichermenge in Terabyte](#terabyte-lizenzierung) (Capacity Licensing). **Einzelprodukte** werden immer nach [Processor Value Units (PVU)](#pvu-lizenzierung) lizenziert.

## IBM Spectrum Protect Einzelprodukte

<Info type="light">

Einzelprodukte werden immer nach [Processor Value Units (PVU)](#pvu-lizenzierung) lizenziert. Zunächst benötigen Sie eines der beiden Grundprogramme (Backup Server) Base Edition oder Extended Edition.

</Info>

### Basis Software

<ArrowList>

- [Spectrum Protect Base Edition](/hersteller/ibm/spectrum-protect/base/)<br />

- [Spectrum Protect Extended Edition](/hersteller/ibm/spectrum-protect/extended/)<br />
  Erweiterter Basis-Software (Backupserver). Für größere Backup-Umgebungen ausgelegt.

</ArrowList>

### Erweiterungsmodule

<ArrowList>

- [Spectrum Protect for Databases](/hersteller/ibm/spectrum-protect/databases/)<br />
  Sichert [Oracle](/hersteller/oracle/), [Microsoft SQL Server](/hersteller/microsoft/sql-server/) und IBM DB2 Datenbanken. Das Vorgängerprodukt heißt TSM for Databases.
- [Spectrum Protect for ERP](/hersteller/ibm/spectrum-protect/erp/)<br />

- [Spectrum Protect for Mail](/hersteller/ibm/spectrum-protect/mail/)<br />
  Sichert HCL Lotus Domino Notes und Microsoft Exchange E-Mail Server. Das Vorgängerprodukt heißt TSM for Mail.
- [Spectrum Protect for SAN](/hersteller/ibm/spectrum-protect/san/)<br />

## Terabyte-Lizenzierung


Spectrum Protect [Suite Entry](/hersteller/ibm/spectrum-protect/suite-entry/), [Suite](/hersteller/ibm/spectrum-protect/suite/) und [Snapshot](/hersteller/ibm/spectrum-protect/snapshot/) werden volumenbasiert nach Terabyte lizenziert. Sie können das Backend-Volumen oder das Frontend-Volumen lizenzieren.


<ArrowList>

- **Backend-Lizenzierung**: Datenvolumen der vom Backupserver gesicherten und verwalteten Daten.
- **Frontend-Lizenzierung**: zu sichernde Daten der Applikationen. Die Berechnung nach Frontend-Daten ist komplizierter, kann jedoch, je nach Anwendungsfall, die günstigere Lizenzierungsalternative sein.

</ArrowList>

<Info>

Einen Sonderfall ist [Spectrum Protect Snapshot](/hersteller/ibm/spectrum-protect/snapshot/). Lizenziert wird immer das Datenvolumen der Snapshots.

</Info>

## PVU-Lizenzierung


[Einzelprodukte](#ibm-spectrum-protect-einzelprodukte) werden abhängig von der Leistung der Backupserver lizenziert. Lizenzmetrik sind [Processor Value Units (PVU)](/glossar/pvu-processor-value-unit-ibm/). Bestimmen Sie zunächst alle physischen Prozessoren aller Backupserver. Weisen Sie dann jedem Prozessor einen modellabhängigen Wert (PVU) zu. Die Summe der PVU müssen Sie lizenzieren.

**Beachten Sie:** Clients/Agents müssen ebenfalls nach dem kumulierten PVU-Wert aller Backupserver lizenziert werden.



Angenommen Sie nutzen zwei Backupserver, die zusammen 800 PVU ergeben. Unter anderem sichern Sie Microsoft Exchange und Oracle Datenbanken. Dafür benötigen Sie die Agents Spectrum Protect for Databases und Spectrum Protect for Mail. Für jeden Agenten müssen Sie ebenfalls 800 PVU lizenzieren, egal auf welcher Hardware die Applikationen laufen.

IBM stellt eine PVU-Tabelle und einen PVU-Rechner im Netz bereit:

<ArrowList>

- [PVU-Tabelle](https://www.ibm.com/software/passportadvantage/pvu_licensing_for_customers.html)
- [PVU-Rechner](https://www-112.ibm.com/software/howtobuy/passportadvantage/pvucalculator/pvucalc.wss)

</ArrowList>
