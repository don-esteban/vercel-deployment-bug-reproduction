## Lizenzüberblick

Eine Project Server CAL wird immer für den Zugriff auf einen Project Serverbenötigt.
Lizenziert wird nach benannten Benutzern ([User CAL](/glossar/user-cal-microsoft/)) oder nach Geräten ([Device CAL](/glossar/device-cal-microsoft/)).
[Project Professional 2021](/hersteller/microsoft/project/professional-2021/) enthält bereits eine CAL.

<ArrowList>

- [Exchange Lizenzierung](/hersteller/microsoft/exchange/lizenzierung/)<br />
  Ausführliche Informationen zu Kauf und Lizenzierung.

- [Exchange Server 2019](/hersteller/microsoft/exchange/server-standard-2019/)<br />
  Jede Exchange Serverinstanz benötigt eine Lizenz.

- [Enterprise CAL](/hersteller/microsoft/exchange/server-enterprise-cal/)<br />
  Upgrade für die Standard CAL mit zusätzlichen Funktionen.

- [CSP](/hersteller/microsoft/csp/)<br />
  Dauerhafte Lizenz (Kauflizenz) ohne Wartung. Keine Mindestbestellmenge. Keine Sonderkonditionen für Behörden.

- [Open Value](/hersteller/microsoft/open-value/)<br />
  Volumenlizenz mit 3 Jahren Software Assurance. Mindestbestellmenge: 5 Lizenzen.

</ArrowList>


## Voraussetzungen zum Betrieb eines lokalen Project Servers

Der Einsatz des Project Servers setzt eine vorhandene Microsoft Infrastruktur voraus. Es werden mindestens ein [Windows Server](/hersteller/microsoft/windows-server/) (zum Betrieb des Project Servers), ein [SharePoint Server](/hersteller/microsoft/sharepoint/server-2019/) (für Datenaustausch) und ein [SQL Server](/hersteller/microsoft/sql-server/) (Datenbank) benötigt. Für In-App-Chat ist [Skype for Business](/hersteller/microsoft/skype-for-business/server-2019/) notwendig. Neben den Server-Programmen sind ebenfalls jeweils passende CALs für die zugreifenden Nutzer und Geräte erforderlich.

Detaillierte Informationen zur Lizenzierung finden Sie im Microsoft Technet: [Software requirements for Project Server 2019](https://learn.microsoft.com/en-us/project/software-requirements-for-project-server-2019)
