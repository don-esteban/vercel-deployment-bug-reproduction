Release Datum des neuen Windows Server 2025 ist das vierte Quartal 2024. Hier erfahren Sie, was heute schon über das neue Release Windows Server vNext bekannt ist. Diese Seite wird laufend mit den neuesten Informationen aktualisiert.


</Intro>

<TOC />


## Windows Server 2025 Editionen

Microsoft hat schon angekündigt, dass sich an den Editionen für Unternehmen gegenüber [Windows Server 2022](/hersteller/microsoft/windows-server/) nichts ändert.
Die beiden Editionen sind funktional identisch und unterscheiden sich lediglich in der unterstützten Anzahl an virtuellen Maschinen:


<ArrowList>

- **Windows Server 2025 Standard**<br />
 Für Server mit geringem Virtualisierungsgrad (maximal 2 Windows Server VMs).

- **Windows Server 2025 Datacenter**<br />
 Für Server mit beliebig hohem Virtualisierungsgrad (unbegrenzte VM Anzahl).

</ArrowList>

## Windows Server 2025 CALs

Auch bei den Zugriffslizenzen wird es keine Änderungen geben. Es werden wieder die folgenden CALs für Benutzer (User CAL) oder Geräte (Device CAL) angeboten:

<ArrowList>

- **Windows Server 2025 CAL**<br />
 Eine Windows Server CAL (Client Access License) wird zum Zugriff auf Windows Server benötigt.

- **Windows Server 2025 External Connector**<br />
 Lizenziert den Zugriff aller externen Nutzer.

- **Windows Server 2025 Remote Desktop Service CAL**<br />
 Lizenziert den Zugriff auf Terminalserver (RDS - Remote Desktop Services).

- **Windows Server 2025 Rights Management Services CAL**<br />
 Lizenziert den Zugriff auf Rechteverwaltung mit Active Directory Rights Management Services (AD-RMS).

</ArrowList>


## Windows Server 2025 Lizenzierung

Windows Server 2025 wird als unbefristete Lizenz angeboten. Für den Zugriff auf den Windows Server 2025 werden Zugrifflizenzen (CALs) benötigt.
Bei der [Windows Server Lizenzierung](/hersteller/microsoft/windows-server/lizenzierung/) sind bislang keine Änderungen bekannt.

<ArrowItem>

Alle Informationen finden Sie auf unserer Seite zur [Windows Server Lizenzierung](/hersteller/microsoft/windows-server/lizenzierung/)

</ArrowItem>

## Bisher bekannte Änderungen

<ArrowList>

- **Hotpatching für alle Editionen**<br />
  Updates und Patches während des laufenden Betriebs, ohne Neustart. Keine Azure Cloud Anbindung erforderlich.
- **Verbessertes Active Directory**<br />
  Verbesserte Leistungs- und Sicherheitsfunktionen. Unterstützung für 32K Domain Controller.
- **Verbesserte NVMe-Speicher Nutzung**<br />
  Die Leistung von NVMe Speichern wurde um bis zu 70% verbessert und die nötige Prozessorlast verringert.
- **Verbesserte Container Unterstützung**<br />
  Die Unterstützung von Conatainern wird flexibler. U.a. werden die Updates von Host und Container entkoppelt.
- **Netzwerk Verbesserungen**<br />
  Neue Funktionen, wie Network ATC und Network HUD. Verbesserte Leistung für SDN Gateways und SDN Multisite.
- **UI-Anpassungen**<br />
  Die meisten UI-Elemente werden von Windows 11 übernommen, darunter die zentrierte Taskleiste.
- Außerdem hat Microsoft angekündigt, auch für die Bereiche Hyper-V & AI weitere Verbesserungen einzuführen.

</ArrowList>

Eine Übersicht der bekannten Änderungen finden Sie auch auf der [Microsoft Seite](https://techcommunity.microsoft.com/t5/windows-server-news-and-best/introducing-windows-server-2025/ba-p/4026374)
und der englischen Ignite Session [What’s New in Windows Server vNext](https://techcommunity.microsoft.com/t5/windows-server-news-and-best/in-case-you-missed-it-windows-server-at-microsoft-ignite/ba-p/3995168).


## Hyper-V als Alternative zu VMware

Nach der Übernahme von VMware durch Broadcom und der Abschaffung der dauerhaften VMware Lizenzierung, positioniert sich Microsoft mit dem Windows Server 2025 weiter als ernstzunehmende Alternative für Virtualisierung. Der Windows Server 2025 erhält zusätzliche Hyper-V Funktionen, die den Einsatz in Rechenzentren vereinfachen.

<ArrowList>

- **GPU-Partitionierung (GPU-P)**<br />
  Mit Windows Server 2025 lassen sich auch Grafikprozessoren (GPU) aufteilen und den VMs zuweisen. Damit sollen auch Migrationen zwischen mehreren Servern unter Beibehaltung von High Availability möglich sein.

- **Workgroup Cluster**<br />
  Neu hinzu kommt auch die Bereitstellung von Workgroup-Clustern, mit zertifikatsbasierter Authentifizierung. Somit ist kein Active Directory mehr für den Einsatz von Windows Server Clustern nötig!

</ArrowList>

## Windows Server 2022

Hier finden Sie alle Informationen und Preise zur aktuellen Windows Server Version 2022. Der Mainstream Support mit Featureupdates läuft noch bis zum 13. Oktober. Erweiterten Support mit Sicherheitsupdates gibt es noch bis 14. Oktober 2031.

<ArrowList>

- **[Windows Server 2022 Datacenter](/hersteller/microsoft/windows-server/datacenter-2022/)**<br />
 Für Server mit beliebig hohem Virtualisierungsgrad (unbegrenzte VM Anzahl).

- **[Windows Server 2022 Essentials](/hersteller/microsoft/windows-server/essentials-2022/)**<br />

</ArrowList>
