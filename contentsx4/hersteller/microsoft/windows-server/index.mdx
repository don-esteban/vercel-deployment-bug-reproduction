<Intro>

 Windows Server 2022 ist die aktuelle Version des Microsoft Server-Betriebssystems. Die Windows Server Editionen Standard und Datacenter unterscheiden sich in den Virtualisierungsmöglichkeiten.
 Lizenziert wird immer nach Cores. Zusätzlich werden Zugriffslizenzen (CAL) für Nutzer und Geräte benötigt. Hier finden Sie alle Informationen zum Kauf von MS Windows Server 2022.


</Intro>

<TOC />


## Windows Server 2022 Editionen

Windows Server ist eine Kauflizenz mit (optionaler) Software Assurance. Die Standard Edition kann alternativ auch als Subscription lizenziert werden.
Mit Windows Server Essentials gibt es auch eine eigenschränkte Edition für kleine Unternehmen.

<ArrowList>

- **[Windows Server 2022 Standard](/hersteller/microsoft/windows-server/standard-2022/)**<br />
 Für Server mit geringem Virtualisierungsgrad (maximal 2 Windows Server VMs).

- **[Windows Server 2022 Datacenter](/hersteller/microsoft/windows-server/datacenter-2022/)**<br />
 Für Server mit beliebig hohem Virtualisierungsgrad (unbegrenzte VM Anzahl).

- **[Windows Server 2022 Essentials](/hersteller/microsoft/windows-server/essentials-2022/)**<br />
 Für kleine Unternehmen mit nur einem Server. Maximal eine CPU und 10 Cores. Nachfolger des Small Business Servers (SBS). Nur als OEM Lizenz.

</ArrowList>


<Info>

Release Datum für den neuen [Windows Server 2025](/hersteller/microsoft/windows-server/2025/) ist das vierte Quartal 2024.

</Info>

## Windows Server CALs

<ArrowList>

- **[Windows Server CAL](/hersteller/microsoft/windows-server/cal/)**<br />
 Eine Windows Server CAL (Client Access License) wird zum Zugriff auf Windows Server benötigt.

- **[Windows Server External Connector](/hersteller/microsoft/windows-server/external-connector/)**<br />
 Lizenziert den Zugriff aller externen Nutzer.

- **[Remote Desktop Service (RDS) CAL](/hersteller/microsoft/windows-server/remote-desktop-services/)**<br />
 Lizenziert den Zugriff auf Terminalserver (RDS, Remote Desktop Services).

- **[Rights Management Services (RMS) CAL](/hersteller/microsoft/windows-server/rights-management-services/)**<br />
 Lizenziert den Zugriff auf Rechteverwaltung mit Active Directory Rights Management Services (AD-RMS).

</ArrowList>

## Core Infrastructure Server (CIS)

Die Microsoft Core Infrastructure Server (CIS) Suite ist ein Bundle aus dem aktuellen Windows Server und der Verwaltungslösung [System Center](/hersteller/microsoft/system-center/). Beide Produkte sind immer mit Software Assurance (SA) ausgestattet, sodass Sie immer die aktuellen Updates sowie Upgrades auf die neuste Version. Das Bundle bietet den Windows Server zusammen mit leistungsstarken Verwaltungstools für eine vereinfachte Administration, reduzierte Supportkosten und erhöhte Sicherheit.

## Windows Server 2022 Funktionen

  Bei der Weiterentwicklung des Windows Servers hat sich Microsoft auf die Schwerpunkte Hybride Cloud, Sicherheit, Applikationsplattform und Hyper-Converged Infrastructure (HCI) konzentriert. Eine Gegenüberstellung der alten Windows Server Versionen mit Windows Server 2019, finden Sie auf unserer [Windows Server Vergleichsseite](vergleich-aller-versionen/).


### Hybride Cloud

  Microsoft gehört zu den weltweit führenden Cloud-Anbietern und investiert massiv in den Ausbau der eigenen Cloud-Infrastruktur [Azure](/hersteller/microsoft/azure/). Wen wundert es daher, dass nun die schrittweise Migration von lokalen Windows Servern in die Azure-Cloud erleichtert wird. Eine wichtige Rolle in diesem Zusammenhang spielt das neue [Windows Admin Center](https://learn.microsoft.com/en-us/windows-server/manage/windows-admin-center/overview), ein browserbasiertes Verwaltungswerkzeug für Windows Server. Hybride Deployments (Azure, Azure Stack, On-Premises, Windows Server 2012 und neuer) lassen sich nun zentral über eine einzige Konsole verwalten. Zu den Möglichkeiten zählen:

<ArrowList>

 - Azure Active Directory Integration
 - Azure Backup Integration
 - Azure Site Recovery Integration
 - Neue Storage Migration Services
 - Einfache Verbindung mit Azure Virtual Networks

</ArrowList>

### Security

  Anforderungen an die Sicherheit von Serverbetriebssystemen steigen kontinuierlich. Microsoft hat vor allem Verbesserungen in den Bereiche Identity Management, Betriebssystemsicherheit und Virtualisierung vorgenommen. Windows Defender Advanced Threat Protection und Exploit Guard sind integriert. Linux VMs werden mit Shielded VMs for Linux geschützt und der Netzwerk-Traffic zwischen virtuellen Maschinen kann verschlüsselt werden.

<ArrowList>

 - Privileged Identity Management
 - Windows Defender Advanced Threat Protection
 - Windows Defender Exploit Guard
 - Shielded VMs for Linux
 - Encrypted Subnets

</ArrowList>


### Applikationsplattform

Moderne Applikationen werden ständig erweitert und verbessert. Wer auf veränderte Anforderungen schnell reagieren kann, macht das Rennen. Anforderung, Implementierung, Test, Rollout. Der Durchlauf dieses Zyklus schrumpft seit Jahren von mehreren Monaten auf Tage oder sogar Stunden. Serverseitig spielen Container ([Mirantis-Docker](/hersteller/mirantis-docker/)) und generell Linux eine immer stärkere Rolle. Diesem Trend kann und will sich auch Microsoft nicht entziehen. Spätestens seit [SQL Server for Linux](/hersteller/microsoft/sql-server/) ist klar, dass Microsoft hier mitspielen wird.

<ArrowList>

 - Unterstützung von Kubernetes
 - Unterstützung von Red Hat open-shift
 - Image-Optimierung für VM- und Container-Deployments auf Core- und Nano-Servern
 - Linux Container direkt auf Windows Host
 - Linux-Tools wie ssh, curl, tar
 - Windows Subsystem for Linux (WSL)

</ArrowList>

### Hyper-Converged Infrastructure (HCI)

  Wie bei der Unterstützung [hybrider Server Deployments](#hybride-cloud) basieren die sichtbaren HCI-Verbesserungen auf dem [Windows Admin Center](https://learn.microsoft.com/en-us/windows-server/manage/windows-admin-center/overview).

<ArrowList>

 - Verwaltung über zentrale Konsole ab Windows 2016 Server
 - Hybrid Cloud integriert
 - Image-Optimierung für VM- und Container-Deployments auf Core- und Nano-Servern
 - Verbesserte Performance

</ArrowList>
