## Anwendungsfälle und mögliche Einsatzszenarien für Microsoft Azure

  Azure hilft Ihnen bei der Beantwortung von Problemen, die ansonsten nur schwer und mit größerem Aufwand zu beheben sind. Wir listen hier eine Auswahl an Lösungen für häufig auftretende Problemfälle.

<ArrowList>

- **Der Windows Server / SQL Server 2008 Support endet und ich bin darauf angewiesen, die Version weiter zu nutzen**<br />
  Viele Kunden sind durch den Einsatz von Software, die nur auf älteren Betriebssystem lauffähig ist, zum Einsatz von Windows Server 2008 gezwungen. Da Microsoft den Support und damit Bugfixes auch für sicherheitsrelevante Schwachstellen zum 14. Januar 2020 endet ([siehe Microsoft News](https://azure.microsoft.com/de-de/blog/announcing-new-options-for-sql-server-2008-and-windows-server-2008-end-of-support/)), ist dieser mittlerweile ein hohes Sicherheitsrisiko für Unternehmen und zwingt diese zum ungewünschten Update auf eine neuere Version. Doch wenn dadurch kritische Software, beispielsweise für Buchhaltung oder CAD, nicht mehr lauffähig ist und ebenfalls aktualisiert werden müsste, stehen viele Unternehmen vor einem Problem.

  Microsoft Azure bietet für diese Fälle die Migration in einen [Windows Server](/hersteller/microsoft/windows-server/) 2008 / 2008 R2 und [Microsoft SQL Server](/hersteller/microsoft/sql-server/) 2008 / 2008 R2 in der Azure Cloud. In der Azure Cloud erhalten Sie die **verlängerten Sicherheitsupdates** kostenlos und haben damit drei weitere Jahre Zeit Ihre Server zu aktualisieren.

- **Die Investitionskosten für die Anschaffung neuer Hardware sind zu hoch?**<br />
  Bei anstehenden Hardware-Investitionen bietet sich die Möglichkeit eine Alternative zum Betrieb der Server im Unternehmen in Erwägung zu ziehen. Dabei stellt sich die Frage, ob die neue Hardware überhaupt benötigt und ausgereizt wird. Meistens kann dieselbe Funktionalität durch einen Azure Cloud-Dienst zu geringeren Kosten abgedeckt werden. Wir beraten Sie hierzu gerne.

- **Zu Spitzenlast-Zeiten sind meine Server ausgelastet?**<br />
  Mit Azure können Sie Auftragsspitzen abfangen, indem Sie dynamisch mehr Rechenpower zur Verfügung stellen, wenn diese benötigt wird. Vermeiden Sie unnötige Investitionen in teure Hardware.

- **Wie kann ich mit Microsoft meine Daten an einem (weiteren) sicheren Ort lagern?**<br />
  Azure bietet einen sicheren Speicherplatz für die Dateiablage und Backups im Unternehmen. Damit können beispielsweise DSGVO-Vorgaben, wie die redundante Sicherung an einem anderen Speicherort, abgedeckt werden.

</ArrowList>


## Wie erhalte ich Zugang zu Azure

 Die Azure Subscription (Azure Abonnement) ist die Basis, die für den Kauf von fast allen weiteren Azure Produkten benötigt wird.
 Auf die Azure Subscription werden alle genutzten Azure-Dienste gebucht, die Abrechnung erfolgt jeweils am Monatsende und richtet sich nach den verwendeten Resourcen der gebuchten Dienste.

 Die Kosten für die Subscription sind somit abhängig von allen darauf gebuchten Diensten und deren jeweiliger Nutzungsdauer.
 Die Azure Produkte sind über die Lizenzprogramme [Cloud Service Provider (CSP)](/hersteller/microsoft/csp/) oder [Enterprise Agreement](/hersteller/microsoft/lizenzierung/) erhältlich.

 Weitere Informationen zur Azure Lizenzierung und der enthaltenen Services finden Sie in unserer [Azure FAQ und Dienstübersicht](/hersteller/microsoft/azure/faq/)

## Der Azure Marketplace

Im Azure Marketplace finden Sie die Softwarelösungen einer Vielzahl von Herstellern, wie [Acronis](/hersteller/acronis/), [Sophos](/hersteller/sophos/), [Veeam](/hersteller/vmware/) und [VMware](/hersteller/vmware/), sowie Open Source Lösungen und weitere Dienste.

Die Software kann hier vorkonfiguriert direkt aus der Azure Cloud bezogen werden, abgerechnet wird meistens monatlich über die [Azure Subscription](#azure-subscription-von-software-express). Diese Modell ist als Pay As You Go (PAYG) bekannt.


### Bring Your Own License (BYOL)

Wenn Sie die Software bereits einsetzen bietet Microsoft spezielle Bring Your Own License (BYOL) Angebote, mit denen Sie Ihre bestehenden Microsoft-Lizenzen mit Software Assurance (beispielsweise [Window Server](/hersteller/microsoft/windows-server/) oder [SQL Server](/hersteller/microsoft/sql-server/)) in der Azure Cloud verwenden können. Dabei ist egal, ob Sie die Lizenzen vorher lokal oder in einer anderen Cloud (z.b. AWS) verwendet haben.

Azure BYOL wird auch für die Software anderer Hersteller in der Azure Cloud angeboten: Die Lizenzen für die Azure Cloud erhalten Sie direkt von uns. Über die Azure Subscription wird zusätzlich der Ressourceneinsatz (CPU, Speicher) in der Azure Cloud berechnet.

**Beispiel:** Das Deployment von [Sophos XG Firewall Lizenzen](/blog/sophos-xg-firewall-kaufoptionen-verstehen/) ist beispielsweise über Azure BYOL möglich. Die Lizenz (erhältlich in [unserem Webshop](/hersteller/sophos/xg-firewall-hardware-appliances/)) schaltet die Software im Azure Marketplace frei. Lediglich für die Cloud-Nutzung nötige Ressourcen, werden über die Azure Subscription abgerechnet.
Gerne besprechen wir mit Ihnen den Einsatz von BYOL. Fragen Sie uns einfach.

## Azure Active Directory

Azure AD wird, im Gegensatz zur Azure Subscription nicht verbrauchsabhängig berechnet. Der Service wird als monatliche Subscription pro verwaltetem Nutzer lizenziert und ist in vielen [Office 365 Plänen](/hersteller/microsoft/office-365/) bereits enthalten.

In welchen Paketen Azure AD enthalten ist und wieviel die einzelnen Pläne kosten, finden Sie auf unserer [Azure Active Directory Seite](/hersteller/microsoft/azure/active-directory/).
