## Lizenzierung

SolarWinds SCM wird anhand der Anzahl der Knoten (Nodes) lizenziert. Ein Knoten ist als Server,
virtuelle Maschine, virtueller Host oder Betriebssysteminstanz definiert. Es spielt keine Rolle,
wie viele Anwendungen oder Datenbanken auf dem Server ausgeführt werden, da der Server
selbst bloß ein Knoten ist.

<ArrowList>

- *Mindestanforderung:* Vierkernprozessor, [Windows Server 2016 oder aktueller](/hersteller/microsoft/windows-server/), SQL-DB.
- *SQL-Datenbank:* [SQL Server](/hersteller/microsoft/sql-server/), Cloud: Amazon RDS, [Azure](/hersteller/microsoft/azure/) SQL-Datenbank.
- Lizenziert wird nach Nodes. (Bspw. Anwendungen, Systeme, Hypervisoren, IaaS, PaaS und SaaS)
- Als Kauflizenz (perpetual) und Subscription (Abonnement)
- Neulizenzen beinhalten bereits 1 Jahr Wartung und Support. In Subscriptions ist diese immer enthalten.
- Jahreslizenzen (Wartung) beinhalten kostenlose Updates und Upgrades, sowie kostenlosen Support.
- Wir listen hier nur Node-Lizenzen bis 1000 Nodes, weitere Lizenzen auf Anfrage.

</ArrowList>


## Funktionen

<ArrowList>

- Umfangreiches Server Konfigurations-Monitoring
- Echtzeiterkennung und Nachverfolgung von System- und Anwendungsänderungen
- Datenbanken auf Konfigurationsänderungen und Änderungen an Benutzerberechtigungen überprüfen
- Überwachung von Serverkonfigurationen anhand von Compliance-Richtlinien
- Offline durchgeführte Änderungen erkennen
- Konfigurationsänderungen über einen Zeitraum hinweg vergleichen
- Problemursachen durch Monitoring schnell herausfinden
- Zentrale Verwaltung von PowerShell-, Bash- und Python-Skripten auf Servern
- Automatisierte Hardware- und Softwarebestands-Verwaltung

</ArrowList>
