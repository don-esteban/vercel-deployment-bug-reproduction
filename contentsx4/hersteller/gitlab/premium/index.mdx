<Intro>

GitLab Premium ist für Entwicklerteams gedacht, die mit mehreren Gruppen an verschiedenen Projekten arbeiten. Beinhaltet eine Entwicklungsplattform mit Versionskontrolle, Jira Development, CI/CD Pipelines Dashboard und priorisiertem Support.

Es gibt 2 Editionen: Premium (Self-Managed, On-Premises) und Premium Plan (SaaS, GitLab Cloud). Der Unterschied liegt nur in der Administration.

</Intro>

<TOC />



## Preise

Gitlab Premium gibt es als Subscription mit einem Jahr Laufzeit. [Lizenziert](/hersteller/gitlab/lizenzierung/) wird pro Benutzer. Enthält [Priority Support](/hersteller/gitlab/lizenzierung/#priority-support). Bei Self-Managed gibt es zusätzliche Supportinhalte.

### GitLab Premium (Self-Managed)

On-Premises oder Public Cloud, Sie verwalten selbst und sind der Host.

## Funktionen und Inhalte

- ChatOps
- Multiple approvers in code review
- Merge approvals
- Code Quality
- Multiple Group
- Jira Issues
- CI/CD Pipelines Dashboard
- CI/CD for external repo
- CI/CD for GitHub
- Issue Boards
- Priority Support
- Multi-project pipeline graphs
- Deploy Boards
- Timed and manual incremental rollout deployments
- Canary Deployments

<Info>

Wenn Funktionen für Lizenzmanagement, Compliance-Richtlinien und mehr Sicherheit benötigt werden, gibt es die [GitLab Ultimate](/hersteller/gitlab/ultimate/) Edition.

</Info>
